  • Q) DB Index?

    A) 조건검색의 속도를 향상시켜주기 위한 기능, 인덱스 조건에 부합하는 데이터가 테이블의 10~15%가 넘지 않도록 설계, 결합인덱스 설정 시 scan결과 데이터가 적은 컬럼부터 순차설정 해야 효율적, where절에 자주 등장하며 범위검색으로 검색되지않는 컬럼

  • Q) DB Index의 는 왜 B-Tree인가?

    A) 항상 정렬된 상태로 특정 값보다 크고 작은 부등호 연산에 문제가 없다. 참조 포인터가 적어 방대한 데이터 양에도 빠른 메모리 접근이 가능하다. 데이터 탐색뿐 아니라, 저장, 수정, 삭제에도 항상 O(logN)의 시간 복잡도를 가진다.

  • Q) DB 조인, 동작원리?

    A) 기본적으로 Inner Join, Outer Join, Cross Join, Self Join / 조인의 방법은 Nested Loop Join, Sorted Merge Join, Hash Join
